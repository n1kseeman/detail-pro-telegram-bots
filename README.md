# Detail Pro — Telegram-бот на Cloudflare

Готовая демонстрационная версия одного Telegram-бота для детейлинг-центра и автомойки. Бот работает на бесплатных Telegram Bot API, Cloudflare Workers и Cloudflare D1 — без ИИ, онлайн-оплаты, постоянно включённого компьютера или отдельного сервера.

## Возможности

- Клиент выбирает одну или несколько услуг, указывает автомобиль, комментарий, до 5 фотографий и телефон через безопасную кнопку Telegram.
- Запись доступна на ближайшие 14 дней с учётом рабочих дней, блокировок и занятых слотов.
- Cloudflare D1 (серверная SQLite-совместимая база) не допускает двойного бронирования: активный слот защищён уникальным индексом базы данных.
- Клиент видит активные и завершённые записи, может отменить активную запись и принять/отклонить предложенное новое время.
- Бот сразу отправляет заявку, фото и кнопки подтверждения, отклонения и переноса в закрытую рабочую Telegram-группу.
- Админ-меню: сегодняшние и активные заявки, поиск по номеру/телефону, расписание на дату, рабочие дни, слоты, блокировки, услуги и статистика.

## Развёртывание 24/7 бесплатно

Нужен бесплатный аккаунт [Cloudflare](https://dash.cloudflare.com/sign-up) и Node.js 20+.

```bash
cd /Users/n1kseeman/Documents/mx
npm install
npx wrangler login
```

Создайте D1-базу:

```bash
npx wrangler d1 create detail-pro-db
```

Скопируйте выданный `database_id` в поле `database_id` файла `wrangler.toml`, затем примените схему:

```bash
npx wrangler d1 migrations apply detail-pro-db --remote
```

Добавьте секреты. Значения токена и группы не попадут в GitHub:

```bash
npx wrangler secret put BOT_TOKEN
npx wrangler secret put ADMIN_GROUP_ID
npx wrangler secret put WEBHOOK_SECRET
npx wrangler secret put MANAGER_IDS
npx wrangler secret put MANAGER_USERNAME
npx wrangler secret put MANAGER_PHONE
```

`MANAGER_IDS` — Telegram ID сотрудников через запятую. Только эти сотрудники могут подтверждать, отклонять, переносить записи и открывать `/admin`. `MANAGER_USERNAME` и `MANAGER_PHONE` необязательны: если они заданы, клиент увидит кнопки связи с менеджером.

Для краткого демонстрационного режима можно добавить секрет `OPEN_ADMIN_ACCESS=true`. Тогда `/admin` и все действия менеджера доступны каждому пользователю. После демонстрации обязательно удалите этот секрет или установите `false`.

Разверните Worker:

```bash
npm run deploy
```

Wrangler выведет адрес вида `https://detail-pro-bot.<ваш-subdomain>.workers.dev`. Установите Telegram webhook, подставив этот адрес, токен и тот же `WEBHOOK_SECRET`:

```bash
curl -X POST "https://api.telegram.org/bot<ТОКЕН>/setWebhook" \
  -d "url=https://detail-pro-bot.<ваш-subdomain>.workers.dev/telegram" \
  -d "secret_token=<WEBHOOK_SECRET>"
```

После этого компьютер можно выключить: Telegram будет отправлять обновления сразу в Cloudflare Worker, а D1 сохранит заявки. Cloudflare D1 доступна на Free и Paid планах. [Документация D1](https://developers.cloudflare.com/d1/)

## Демо-сценарий

1. Откройте бота и отправьте `/start`.
2. Нажмите «Записаться», отметьте услуги, заполните данные и поделитесь контактом.
3. После отправки запись получит статус «Ожидает подтверждения».
4. Бот отправит заявку в рабочую группу; подтвердите её, отклоните с причиной или предложите свободное время.
5. В админ-меню можно быстро настроить график, слоты и заблокировать день/отдельное время.

### Заявки в рабочую группу

Добавьте этого же бота в нужную группу, отправьте в ней команду `/chatid` и скопируйте ответ (обычно начинается с `-100`). Перед развёртыванием внесите его в секрет `ADMIN_GROUP_ID`. Кнопки обработки заявок работают только у сотрудников из `MANAGER_IDS`.

## Управление для менеджеров

В личном чате с ботом менеджер из `MANAGER_IDS` отправляет `/admin` и получает кнопки для просмотра активных записей, расписания, рабочих дней, слотов, услуг, статистики и ошибок доставки.

- `/find НОМЕР_ЗАЯВКИ_ИЛИ_ТЕЛЕФОН` — поиск записи;
- `/block ГГГГ-ММ-ДД [ЧЧ:ММ]` и `/unblock ...` — блокировка даты или отдельного времени;
- `/slot add ЧЧ:ММ` и `/slot remove ЧЧ:ММ` — добавление/отключение слота;
- `/service add Название | Описание` — добавить услугу;
- `/service edit ID | Название | Описание` — изменить услугу;
- `/service on ID` и `/service off ID` — показать или скрыть услугу.

При выборе «Отклонить» в сообщении заявки менеджер указывает следующим сообщением причину, и она отправляется клиенту.

## Надёжность и восстановление

Worker сохраняет обработанные Telegram updates и игнорирует повторную доставку одного события. Ошибки отправки в Telegram записываются в таблицу `delivery_failures` и доступны в `/admin` → «Ошибки доставки». Для восстановления данных используйте резервные возможности Cloudflare D1 и регулярно экспортируйте критичные записи из D1.
